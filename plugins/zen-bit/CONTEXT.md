# BandsInTown Plugin Context — /plugins/zen-bit

> **Responsibility:** Sync and display Bandsintown events with SEO optimisation and SWR caching.
> **Current API version:** `zen-bit/v2` (namespace `/wp-json/zen-bit/v2/`)
> **Plugin version:** 3.1.0

---

## Logic Flow

1. **API Integration:** Consumes the Bandsintown REST API using `zen_bit_artist_id` (or
   `zen_bit_artist_name` when set) and `zen_bit_api_key` configured in the admin panel.

2. **Data Ownership:** This plugin is the single owner of event data. It filters events
   without dates, validates required fields, and sanitises all content before delivery.
   The React frontend is a pure consumer — it renders what this plugin delivers via REST.

3. **REST Endpoints (`/wp-json/zen-bit/v2/`):**

   | Method | Path | Description |
   |--------|------|-------------|
   | `GET`  | `/events` | Paginated event list (ZenBitEventListItem) |
   | `GET`  | `/events/{event_id}` | Full event detail (ZenBitEventDetail) |
   | `GET`  | `/events/schema` | JSON-LD list (MusicGroup + MusicEvent[]) |
   | `GET`  | `/events/{event_id}/schema` | JSON-LD single event |
   | `POST` | `/admin/fetch-now` | Force refresh (JWT or Cookie admin required) |
   | `POST` | `/admin/clear-cache` | Clear all cache (JWT or Cookie admin required) |
   | `GET`  | `/admin/health` | Health status JSON (JWT or Cookie admin required) |

4. **Caching — SWR (Stale-While-Revalidate):**
   - Transients expire per configurable TTLs (`zen_bit_ttl_upcoming`, `zen_bit_ttl_detail`,
     `zen_bit_ttl_past`).
   - On cache MISS: acquires an anti-stampede lock (30 s) then fetches externally.
   - On lock contention: serves a persistent fallback from `wp_options` (never empty).
   - Default TTLs: upcoming 6 h, detail 24 h, past 7 d.

5. **Sitemap:** `/sitemap-events.xml` — dynamically generated using canonical URLs, cached 6 h.

---

## Rules

1. **Performance:** CSS/JS assets MUST only be enqueued when the `[zen_bit_events]`
   shortcode is present on the page (checked via `Zen_BIT_Shortcode` during the `wp_enqueue_scripts` hook).

2. **SEO — canonical URLs:** ALL URLs (sitemap `<loc>`, API `canonical_url`, schema
   `url` field) MUST be generated by `Zen_BIT_Normalizer::build_canonical_path()`:
   ```
   /events/{yyyy-mm-dd}-{slug}-{event_id}
   ```
   Never use `/events/{event_id}` directly in any consumer — that is the short-form
   alias and is NOT the URL the React SPA renders.

3. **Schema:** The shortcode `[zen_bit_events_schema]` injects `ld+json` for SSR pages.
   The REST endpoint `/events/schema` serves the same data for the headless frontend.

4. **Architecture:** Strict Singleton. Bootstrap occurs in `zen-bit.php`.
   The admin class `Zen_BIT_Admin` is loaded conditionally within `load_dependencies()`.

5. **Double-fetch prevention:** `get_event()` MUST search the SWR pool cache before
   calling `fetch_from_bandsintown()`. The lookup order is:
   - Tier 1: dedicated detail transient for `event_id`.
   - Tier 2: upcoming pool transient (free lookup).
   - Tier 3: past pool transient (free lookup).
   - Tier 4: live HTTP fetch (only if all tiers miss).

## Options Reference

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `zen_bit_artist_id` | string | `15619775` | Bandsintown numeric artist ID |
| `zen_bit_artist_name` | string | `` | Artist name (takes priority over ID when set) |
| `zen_bit_api_key` | string | `` | Bandsintown app_id |
| `zen_bit_default_days` | int | `365` | Default day range for queries |
| `zen_bit_ttl_upcoming` | int | `21600` | Upcoming events cache TTL (s) |
| `zen_bit_ttl_detail` | int | `86400` | Event detail cache TTL (s) |
| `zen_bit_ttl_past` | int | `604800` | Past events cache TTL (s) |
| `zen_bit_enable_schema` | bool | `true` | Enable `/events/schema` endpoint |
| `zen_bit_include_raw_debug` | bool | `false` | Include raw BIT response in detail (dev only) |

## Key Files

| File | Responsibility |
|------|---------------|
| `zen-bit.php` | Bootstrap, Singleton, REST route registration, activation/deactivation |
| `includes/class-zen-bit-api.php` | HTTP fetch, SWR pool, all REST endpoint callbacks, JWT auth |
| `includes/class-zen-bit-cache.php` | SWR logic, anti-stampede lock, health tracking, TTL helpers |
| `includes/class-zen-bit-normalizer.php` | Raw → ZenBitEventListItem / ZenBitEventDetail / JSON-LD |
| `includes/class-zen-bit-sitemap.php` | `/sitemap-events.xml` generation using canonical URLs |
| `admin/class-zen-bit-admin.php` | WP Admin UI, settings registration, admin asset management |
| `public/css/zen-bit-public.css` | Shortcode styles (loaded conditionally) |

---

*Keep events synchronised and cache TTLs respected to avoid Bandsintown API rate limits.*
*When in doubt about a URL, always delegate to `Zen_BIT_Normalizer::build_canonical_path()`.*
